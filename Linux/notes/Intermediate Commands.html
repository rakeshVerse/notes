Intermediate Commands

  • env: output the list of all environment variables, we can also use printenv which return same o/p

  • history: show command history (list of previously executed commands)

  • head: output first part of the files. By default it output first 10 lines but we can use -n option to specify number of lines we want to output
    Usage: head [option] [file_name]
    Eg: head myFile.txt , head myFile.txt -n 50

  • tail: output last part of the files. By default it output last 10 lines. We can specify number of lines we want to output using -n option
    Usage: tail [option] [file_name]
    Eg: tail myFile.txt , tail -n 50 myFile.txt
    Option -f : -f output last part of the files then it does not stop there, it keep listening for the new update to the file and output it. It is very useful for system monitoring. It is usually used to monitor system files to see new updates to the file:
    tail -f /var/log/system.log

  • cat: concatinate files and print their entire content on the standard output, so we can print the content of multiple files
    Usage: cat [option] [file_name]
    Eg: cat demo.txt
    cat -n demo.txt # -n option print the line numbers
    cat demo.txt recent_pkgs.txt # print the content of 2 files to the standard output

  • date: output current date and time

  • Redirecting Standard Output: result return by commands outputs on standard output (Terminal) but we redirect/write the result of commands to a file.
    If file exists then it overwrite or append the result to the file.
    If file doesn't exists then it first creates the file then write into it:

    • Overwrite(>): it overwrites the file if the file already exists. All the contents of the file will replaced by the new content:
      Usage: [command] > [file_name]
      Eg: pwd > demo.txt
      cat demo.txt recent_pkgs.txt > concat.txt # concatinate the contents of multiple files and redirect/write it to a single file concat.txt (instead of printing it on the standard output)
    • Append(>>): it appends the result at the end of the file:
      Usage: [command] >> [file_name]
      Eg: pwd >> demo.txt
      cat demo.txt >> list.txt # append the contents of demo.txt to list.txt

  • less: like cat less is also used to view content of the files except it is more useful for large files as it open files on a seperate wizard & doesn't just print entire content of the file.
    Instead it prints lines as per window size at a time which makes it more readable specially for large files. We can scroll downwards and upwards. We can also search for specific words in the files.
    Shortcuts:

    • g - show start of the file
    • G - show end of the file
    • q - quit the wizard
    • /[word] - search [word] in the file
    • space - scroll page down
    • b - scroll page up

  • echo: print the given string to the standard output. We can also write strings to the file instead which is very useful when we want to write something to a file very quickly.
    So, instead of creating or opening the file first then edit it using editor, we can just use echo . It is very userful to make small changes to files quickly.
    Eg:
    echo "hello" # prints hello on the screen
    echo "USERNAME=bolt" > config.cnf # writes given string to the file
    echo "EMAIL=abc@xyz.com" >> config.cnf # appends given string to the file

  • wc: print number of lines, number of words and size of file (in bytes) respectively
    Usage: wc [option] [file_name]
    Eg: wc list.txt # prints - 87 594 6450 list.txt
    wc list.txt demo.txt # we can also print the count of more than one file
    wc -l list.txt # -l prints no. of lines only

  • Pipe: We know how to take output of a command and redirect/write it to a file using '>' & '>>'.
    With pipe (|) we can pass the output of one command as input to the another command.
    Usage: [command] | [command]
    Few Examples:
    Suppose, we want to the count no. of lines return by ls -a command. One way is to redirect output of ls -a to a file then see the no. of lines by running wc on that file.
    The other way is using Pipe, where we can directly pass the output of ls -a to wc and see the no. of lines return:
    ls -a | wc # output - 54 54 583
    We can also concatinate multiple files and see their count:
    cat list.txt demo.txt | wc
    We can also redirect the output of above command to a file:
    cat list.txt demo.txt | wc > count.txt

  • sort: output the file contents sortedly. It doesn't change the file. Usage: sort [option] [file_name]

    • text sort: sort text alphbeticly:
      sort textfile.txt
      sort test.txt testsort.txt # sort multiple files
    • numeric sort: sort numbers: sort -n numfile.txt
    • reverse sort: sort in desending order:
      sort -r textfile.txt # reverse sort text file
      sort -nr numfile.txt # reverse sort numeric file
    • unique get only unique values
      sort -u testfile.txt # unique sort text file with
      sort -nu numfile.txt # unique sort numeric file
    • sort with pipe: get the contents of multiple files and sort them:
      cat file1.txt file2.txt | sort
      Also, if we want to see the unique values inside a file, first unique sort the file then pass the output to wc:
      sort -u testfile.txt | wc -l # text file
      sort -nu numfile.txt | wc -l # numeric file
    • ignore case: by default sort is case-sensative, to make it insensitive use -f option.
      If we have two similar words with different cases (apple & Apple) they will be considered as two unique values.
      But if we use -f, those two values will be consider similar:
      sort -u textfile.txt # will output both apple & Apple as they are unique because of different case
      sort -uf textfile.txt # will output apple only as -f makes the output case-insensitive

  • uniq: remove adjacent duplicate values only
    Usage: uniq [option] [file_name]
    Eg: uniq testsort.txt

    If we sort the file first (as sort will group all the duplicate values together) then pass output to uniq then all the duplicate values will be removed:
    sort testsort.txt | uniq which is silmilar to sort -u testsort.txt

    • -d: output only adjacent dulplicate values i.e. omit values appear only once adjacently: uniq -d testsort.txt
      We can sort the file first (as sort will group all the duplicate values together) then pass the output to uniq to get all the duplicate values only once: sort testsort.txt | uniq -d

    • -u: output adjacent non-duplicate values: uniq -u testsort.txt
      We can use it with sort to get non-duplicate values only, which omit all the values appear more than once: sort testsort.txt | uniq -u

    • -c: count the no. of times the value has appear in the file adjacently: uniq -c testsort.txt
      With sort (as sort groups duplicate values together): sort testsort.txt | uniq -c
      We can pass the o/p of above command using pipe to sort the count numericly: $ sort testsort.txt | uniq -c | sort -n

  • xargs: o/p of a command is used as the i/p of another command: command1 | xargs command2. We have already seen that we can do the same using pipe only for example: ls | sort. Notice, sort has no arguments, here sort will take the o/p of ls as input/argument. But not all commands work like that. Some commands, expects arguments to be passed explicitly otherwise it will throw 'missing operand' error. For such commands we need to pass o/p of one command as space seperated arguments to the another command. xargs takes the o/p of command and turn it into list of space seperated arguments that other commands would accept.
